# Практика 32 — AI-интеграция (потоковый чат + RAG)

## Цель
Сделать чат-приложение с потоковым ответом (Server-Sent Events) и поиском по загруженным документам (мини-**RAG** — Retrieval-Augmented Generation).

## Что реализовано
### Сервер
| Метод | Путь | Описание |
|---|---|---|
| POST   | `/api/chat`              | Stream-ответ через SSE (`{ conversationId, message }`) |
| GET    | `/api/conversations/:id` | История диалога |
| POST   | `/api/documents`         | Загрузить документ для RAG (`{ name, text }`) |
| GET    | `/api/documents`         | Список загруженных документов |
| DELETE | `/api/documents/:id`     | Удалить документ |

### Два режима работы
- **Без `OPENAI_API_KEY`** — встроенный mock: стримит ответ по словам, узнаёт темы (`graphql`, `rabbit`, `jwt`, `docker`, …) и подмешивает фрагменты из загруженных документов.
- **С `OPENAI_API_KEY`** — реальные потоковые ответы от OpenAI Chat Completions API.

### Мини-RAG
Наивный поиск по ключевым словам: для каждого документа считается, сколько слов запроса встретилось в тексте; топ-2 идут в контекст модели. В реальном проекте сюда подключают **pgvector** + эмбеддинги (`text-embedding-3-small`).

### Клиент
- Чат-интерфейс с пузырями user / assistant / context.
- Поле ввода + кнопка отправки.
- Кнопка «📎 Загрузить документ» — поддерживает `.txt`/`.md`.
- Streaming-парсер SSE (`data:` события).

## Файлы
| Файл | Назначение |
|---|---|
| `server.js`          | Express + SSE + mock/OpenAI streaming + RAG |
| `public/index.html`  | Чат-интерфейс |
| `public/style.css`   | Тёмная тема, пузыри сообщений |
| `public/app.js`      | SSE-парсер, загрузка документов |

## Как запустить
```bash
cd practice-32-ai
npm install
npm start                  # mock-режим
# или с реальным OpenAI:
OPENAI_API_KEY=sk-... npm start
```
Открыть **http://localhost:3000**.

## Сценарий проверки
1. Спросить «Что такое GraphQL?» — mock ответит по теме.
2. Нажать «📎 Загрузить документ», выбрать любой `.txt` (например, README этой практики).
3. Спросить вопрос по содержимому документа — над ответом появится подпись «📚 Контекст: имя-файла», а ответ начнётся со ссылки на него.
