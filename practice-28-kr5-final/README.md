# ⭐ Контрольная работа №5 — Итоговый проект «Автосервис Kanban & Collab»

Этот проект является финальной точкой сборки всего курса («Фронтенд и бэкенд разработка»). Он объединяет все полученные знания из **практик 1–32** и представляет собой полноценное отказоустойчивое, масштабируемое PWA-приложение для совместной работы в реальном времени.

За основу взята архитектура **КР4 (практика 24)** и трансформирована в **Вариант 5: Менеджер задач автосервиса (Kanban-доска с real-time синхронизацией и Web Push уведомлениями)**.

## 🏗️ Архитектура системы

Проект разворачивается одной командой в полностью изолированном сетевом окружении **Docker Compose**:

```
                          🌐 Браузер Клиента
                                 │
                                 ▼  http://localhost:8080 (Входная точка)
                        ┌───────────────────┐
                        │    Nginx (LB)     │  ← Отдаёт React SPA (статику)
                        └────────┬──────────┘
                                 │
             ┌───────────────────┼────────────────────┐
             │                   │                    │
    /api/* + /api-docs   /socket.io/* (WS)           /sw.js
    (Round Robin)        (Clustered Pub/Sub)          (Оффлайн-кэш PWA)
             │                   │
             └─────┬─────────────┘
                   ▼
    ┌──────────────────────────────────────┐
    │        upstream tasks_backend        │
    │  cars-backend-1 (3000)               │
    │  cars-backend-2 (3000)               │
    │  cars-backend-3 (3000)               │
    └────────────┬─────────────────────────┘
                 │ pg + redis (по DNS)
        ┌────────┴────────┐
        ▼                 ▼
   ┌─────────┐      ┌──────────────────┐
   │Postgres │      │      Redis       │
   │  :5432  │      │      :6379       │
   └─────────┘      ├──────────────────┤
                    │ • Кэш доски      │
                    │ • Socket.IO Pub/Sub│
                    │ • Push-подписки  │
                    └──────────────────┘
```

## 🧰 Интегрированный стек технологий (Практики 1–32)

1.  **Frontend (React SPA)**:
    *   **React 18** — динамический интерфейс, хуки состояния, оптимистичные обновления Kanban-доски.
    *   **HTML5 Drag-and-Drop API** — плавный перенос карточек задач между колонками.
    *   **Sass/SCSS** — премиальная тёмная тема (`hsl` цвета, размытие панелей, glassmorphism).
    *   **Axios Interceptors** — автоматическое обновление JWT токенов (access/refresh) без прерывания сессии пользователя.
    *   **Socket.IO Client** — мгновенные оповещения о действиях других мастеров.
    *   **PWA**:
        *   `manifest.json` — установка приложения на рабочий стол.
        *   `sw.js` — Service Worker с кэшированием (Cache First) статики для работы оффлайн.
        *   **Web Push Notification API** — системные пуши на уровне ОС.
2.  **Backend (Node.js & Express)**:
    *   **Express 4** — защищённый REST API.
    *   **JWT + RBAC Middleware** — авторизация с разделением ролей (`user` / `seller` / `admin`).
    *   **Nanoid & Bcryptjs** — безопасная генерация идентификаторов и хеширование паролей.
    *   **Web-Push** — отправка push-уведомлений по протоколу VAPID.
    *   **Swagger-JSDoc** — интерактивная документация API на `/api-docs`.
3.  **Базы данных и оптимизация (KR4)**:
    *   **PostgreSQL 16** — постоянное хранение пользователей, задач автосервиса и связей.
    *   **Redis 7**:
        *   **Кэширование**: GET-запросы кэшируются на 10 минут. Кэш сбрасывается мгновенно при мутациях (POST/PATCH/DELETE).
        *   **Socket.IO Cluster**: `@socket.io/redis-adapter` синхронизирует WebSockets между backend-инстансами.
4.  **Контейнеризация и оркестрация**:
    *   **Docker & Docker Compose** — автоматическая сборка фронтенда, бэкенда, БД, Redis, балансировщика Nginx и миграций одной командой.
    *   **Nginx** — Round-Robin балансировка запросов между 3 экземплярами бэкенда, раздача SPA статики, проксирование WebSocket.

## 🔐 Полная матрица ролевого доступа (RBAC)

Для обеспечения безопасности в Kanban-доске настроены следующие права:

| Роль | Описание роли | Просмотр доски | Перенос статуса (Drag-n-Drop) | Создание задач | Редактирование деталей | Удаление задач | Администрирование пользователей |
|---|---|---|---|---|---|---|---|
| **driver** (`user`) | Владелец автомобиля | ✅ Да | ✅ Да (своих задач) | ❌ Нет | ❌ Нет | ❌ Нет | ❌ Нет |
| **mechanic** (`seller`) | Мастер-приёмщик | ✅ Да | ✅ Да | ✅ Да | ✅ Да | ❌ Нет | ❌ Нет |
| **admin** (`admin`) | Управляющий сервиса | ✅ Да | ✅ Да | ✅ Да | ✅ Да | ✅ Да | ✅ Да |

## 🚀 Запуск проекта одной командой

### Требования
*   Установленные **Docker** и **Docker Compose**.

### Шаги запуска
1.  Перейдите в директорию работы:
    ```bash
    cd practice-28-kr5-final
    ```
2.  Запустите контейнеры:
    ```bash
    docker compose up --build
    ```
3.  Откройте в браузере: **[http://localhost:8080](http://localhost:8080)**.
4.  Интерактивный Swagger API доступен по адресу: **[http://localhost:8080/api-docs](http://localhost:8080/api-docs)**.

### 👤 Предустановленные тестовые аккаунты (Seed)

## 🔍 Сценарий приёмки (Как проверить работоспособность)

1.  **Проверка PWA**: Откройте `http://localhost:8080`. Браузер предложит установить приложение как нативное.
2.  **Оффлайн-режим**: В DevTools перейдите во вкладку *Network* и выберите *Offline*. Перезагрузите страницу — интерфейс SPA продолжит загружаться благодаря Service Worker!
3.  **Совместная работа в реальном времени (WebSockets)**:
    *   Откройте две вкладки `http://localhost:8080` (одну в Chrome, вторую в режиме инкогнито или в другом браузере).
    *   Войдите под разными пользователями.
    *   Перетащите карточку «Замена свечей зажигания» из колонки «К выполнению» в «В работе».
    *   Во второй вкладке карточка переместится **мгновенно и плавно**, без перезагрузки страницы!
4.  **Проверка кэша Redis и балансировки**:
    *   При обновлении доски в правом верхнем углу интерфейса отображается индикатор:
        `⚡ Redis cache · cars-backend-2` или `🗄️ PostgreSQL · cars-backend-1`.
    *   Нажмите `↻ Обновить`. Вы увидите переключение источника (`cache` ↔ `server`) и чередование обслуживающих инстансов (`backend-1/2/3`), что подтверждает идеальную работу балансировщика Nginx и Redis-кэша.
